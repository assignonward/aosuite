GMP Integer Storage

GMP integers are stored in BCD
*Binary Coded Decimal, where each 4 bits represents one number 0-9. When viewed in a hex editor, BCD numbers may be read as base 10 numbers directly.
with a terminating nybble of 0xE for a positive number and 0xF for a negative number.
*Nybbles with value 0xA, 0xB and 0xC indicate corrupted or otherwise erroneously recorded data. If 0xD appears immediately behind 0-9 values that is also an error, 0xD is used to pad the unused nybble in the final byte of a GMP encoding when the number has an even number of digits, plus the sign/terminator nybble, plus the 0xD padding nybble.
The ratio type is stored as two GMP integers back-to-back, but both integers are padded, if needed, to end on byte boundaries.

GMP integers (and the library used to work with them) can store and calculate with arbitrarily large numbers. For purposes of Assign Onward, only integers and ratios of integers are used, to explicitly avoid the ambiguity that can come from floating point arithmetic.
